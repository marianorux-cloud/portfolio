# Selective Adoption Implementation Guide: Tremor for auto-lola v2

## Overview

Selective Adoption means using Tremor **only** for:
1. **Reporting Page** - Migrate all Recharts components to Tremor's pre-built chart components
2. **Form Blocks** - Apply Tremor Blocks to forms as needed (deferred)

Keep all other UI components using the existing Shadcn/Radix stack. This minimizes risk while gaining Tremor's specialized dashboard capabilities.

---

## Design System Normalization

**Goal**: Ensure Tremor components use the same design tokens as the existing project for visual consistency.

### Current Project Design System
- **Tailwind v4** with CSS-first config (`@theme inline` in `src/index.css:263-298`)
- **OKLCH-based colors** for charts: `--chart-1` through `--chart-5` (lines 34-38, 82-86)
- **Dark mode**: Uses `.dark` class (lines 61-107)
- **Accent colors**: Multiple accent themes via CSS classes (lines 358-476)
- **Base colors**: Multiple base themes via CSS classes (lines 478-906)
- **Density system**: `compact`, `default`, `comfortable` (lines 110-138)

### How Tremor Integrates
Tremor v4 is built for Tailwind v4 and uses standard Tailwind theme variables. The project's `@theme inline` block already maps:

```css
--color-chart-1: var(--chart-1);  /* line 286 */
--color-chart-2: var(--chart-2);  /* line 287 */
/* ... through chart-5 */
```

Tremor charts use these `chart-*` variables automatically. No additional mapping needed.

### Verification Checklist
- [ ] Tremor components render with project's OKLCH chart colors
- [ ] Dark mode switches correctly (`.dark` class)
- [ ] Density changes (compact/default/comfortable) don't break Tremor layouts
- [ ] Accent color changes propagate to Tremor components

## Step 3: Form Blocks (Deferred - Apply as Needed)

Tremor Blocks provide pre-built form layouts at [blocks.tremor.so/form-layouts](https://blocks.tremor.so/form-layouts).

### Integration Strategy
1. **Keep existing Shadcn/Radix form inputs** (they're already working)
2. **Use Tremor Blocks for layout only** - copy the outer layout structure
3. **Replace inner content** with existing form components

### Example Integration
```tsx
// Tremor Block provides the layout shell
<div className="tremor-block-form-layout">
  {/* Existing Shadcn/Radix form components go here */}
  <ReservationForm />
</div>
```

### When to Apply
- New forms: Start with a Tremor Block layout
- Existing forms: Only apply if a redesign is already planned
- Complex forms: Tremor's grid/list blocks can improve organization
