# Reporting Page Analysis & Enhancement Plan

## Current State: ReportingPage.tsx

### Location
`src/features/reporting/ReportingPage.tsx`

### Data Source
Uses `useData()` hook from `@/context/useData` providing:
- `autos`: Auto[] - vehicle inventory

**Reservation Status Logic** (`src/lib/status.ts`):
- `upcoming` - startDate is in the future
- `active` - Today is within startDate-endDate
- `complete` - endDate is in the past
- `cancelled` - (status field set explicitly)

### Denormalized Fields (`src/lib/denormalize.ts`)
- `renterName` - Copied from Renter.name at load time
- `autoLabel` - `color + vehicle` (e.g., "Blue Toyota Camry")
- `autoShortLabel` - `color + vehicle.split(' ')[0]` (e.g., "Blue Toyota")
- `reservationLine` - Active reservation summary for Auto display

## Data Gaps & Missed Opportunities

| Category | Missing Data | Business Impact |
|----------|--------------|-----------------|
| **Revenue** | No breakdown by auto/renter | Can't identify top revenue sources |
| **Utilization** | No reservation duration tracking | Can't calculate vehicle usage % |
| **Geographic** | Pickup/dropoff locations tracked but not reported | Can't optimize locations |
| **Client History** | No reservation counts per renter | Can't identify loyal customers |
| **Fleet Health** | No maintenance tracking | Can't predict breakdowns |
| **Seasonality** | No date filtering | Can't analyze trends |
| **Rate Analysis** | dailyRate not used in reports | Can't analyze pricing effectiveness |

## Enhancement Options

### Option A: Date Filtering
Add a date range picker to filter reservations for revenue charts and metrics.

**Calculation:** Filter reservations where startDate falls within selected range.
**Impact:** Enables historical analysis, trend identification.

### Option B: Renter Performance Dashboard
Top renters by reservation count and total revenue.

**Calculation:** Group reservations by renterId, count and sum totalPrice (excluding cancelled).
**Impact:** Identifies VIP customers, loyalty programs.

### Option C: Auto Utilization Report
Show which vehicles are most/least used (reservation days vs total days).

**Calculation:** For each auto, count days in active reservations / total days in period.
**Impact:** Fleet maintenance planning, pricing optimization.

### Option D: Geographic Analysis
Pickup/dropoff location distribution (MIA/FLL/Other breakdown).

**Calculation:** Count reservations by pickupLocation and dropoffLocation.
**Impact:** Location optimization, staffing decisions.

### Option E: Revenue Breakdown
Revenue by auto, renter, or location.

**Calculation:** Sum totalPrice grouped by autoLabel/renterName/pickupLocation.
**Impact:** Understand revenue sources, negotiation leverage.

### Option F: Rate Analysis
Average daily rate, price trends over time.

**Calculation:** Average dailyRate across all reservations, trend by month.
**Impact:** Pricing strategy, market positioning.

---

## Recommendation

**Implement Option A (Date Filtering) + Option B (Renter Performance)** as the highest-value combination.

Date filtering provides the foundation for all other metrics. Renter performance is a natural next step that directly impacts business growth.

1. Add DateRangePicker component to page header
2. Create filtered data selectors using the date range
3. Update revenue chart and metrics to use filtered data
4. Add "Top Renters" card showing top 3 by revenue
5. Add "Revenue Breakdown" pie showing by location (MIA/FLL)
