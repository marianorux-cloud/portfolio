# Reservation Form Fixes Plan

## Issues to Address

1. **Date Range Picker - Popover closes after selecting start date**: The picker closes immediately after selecting the first date because react-day-picker returns `{ from: date, to: date }` on single click.

2. **Daily Rate Validation**: Form doesn't display validation error when daily rate is 0 because form uses `mode: 'onSubmit'` by default.

3. **Error Styling**: Label error styling doesn't appear until form submission attempt.

4. **Label-to-Field Spacing**: Needs tighter spacing (~0.125rem instead of 0.25rem).

5. **Error Message Spacing**: Needs reduced spacing between field and error message.

6. **Client vs Auto placeholder boldness**: Different visual weight between select placeholders due to spacing mechanism differences.

---

## Root Cause Analysis

### 1. Date Range Picker - Popover closes after start date selection
- **Location**: `src/components/DateRangePicker.tsx`, lines 26-36
- **Root Cause**: 
  - react-day-picker's `mode="range"` returns `{ from: date, to: date }` when user clicks a single date
  - The condition `range?.from && range?.to` is true immediately on single click
  - This triggers `setOpen(false)` prematurely, closing the popover before user can select an end date
- **Expected behavior**: Range picker should keep popover open between selecting start and end dates

### 2. Daily Rate Validation Not Displaying
- **Location**: `src/components/ReservationForm.tsx`, line 58 and `src/lib/schemas/reservationSchema.ts`
- **Root Cause**: 
  - Schema uses `z.number().positive()` requiring `dailyRate > 0`
  - Default value is `0`, which fails validation
  - Form uses default `mode: 'onSubmit'` - errors only show after form submission attempt
  - `FormMessage` component works correctly but won't show errors until form is submitted
- **Expected behavior**: On form submit, show "Required" error for daily rate = 0

### 3. Error Styling for Required Fields
- **Location**: `src/components/ui/form.tsx`, lines 89-103
- **Root Cause**: 
  - `FormLabel` correctly applies `text-destructive` when `error` exists
  - Same issue as #2 - validation not triggered until submit
- **Expected behavior**: Label should show error styling immediately when validation fails

### 4. Label-to-Field Spacing (Component-wide)
- **Location**: `src/components/ui/form.tsx`, line 83
- **Root Cause**: `FormItem` uses `space-y-1` (0.25rem) as the sole spacing mechanism
- **Expected behavior**: Slightly tighter spacing between label and field (~0.125rem) for compactness

### 5. Error Message Spacing
- **Root Cause**: Same `space-y-1` in FormItem creates gap between field and error message
- **Expected behavior**: Reduce spacing between control and error message

### 6. Client vs Auto placeholder text boldness
- **Location**: `src/components/Select.tsx`, line 52 and `src/components/ui/form.tsx`, line 83
- **Root Cause**: 
  - `Select` wrapper uses `gap-1.5` (flex with gap) while nested inside `FormItem` which uses `space-y-1`
  - Different spacing mechanisms create visual density differences
  - The parent FormItem's `space-y-1` may not be applying correctly to the nested Select wrapper
- **Expected behavior**: Both placeholders should have consistent visual weight

## Implementation Plan

### Fix 1: Date Range Picker - Keep open after first selection
```typescript
// src/components/DateRangePicker.tsx
const handleSelect = (range: DateRange | undefined) => {
  if (range?.from && range?.to && range.from.getTime() !== range.to.getTime()) {
    // Both dates selected and different - close popover
    onChange({ start: range.from, end: range.to })
    setOpen(false)
  } else if (range?.from) {
    // Only start selected (or same day clicked) - keep open
    onChange({ start: range.from, end: null })
    setOpen(true) // Explicitly keep open
  } else {
    onChange({ start: null, end: null })
  }
}

// Also add min={1} to Calendar to prevent same-day ranges
<Calendar
  mode="range"
  selected={selectedRange}
  onSelect={handleSelect}
  numberOfMonths={2}
  min={1} // Require at least 1 night between dates
/>
```
