# Plan: Add Supabase Auth & Deploy to Vercel

## Understanding
- Current app: React/Vite frontend with React Router, static JSON data, no backend
- User wants: Supabase Auth for login, deploy to Vercel to make it live

## Implementation Plan

### Phase 1: Supabase Setup
1. Create Supabase project and configure auth settings
2. Add `@supabase/supabase-js` dependency
3. Create `src/lib/supabase.ts` - Supabase client initialization
4. Create `src/context/AuthContext.tsx` - Auth provider with session/user state
5. Create `src/features/auth/login-page.tsx` - Login form with email/password
6. Create `src/features/auth/auth-provider.tsx` - Wrapper combining Supabase and app context

### Phase 2: Integrate Auth into App
1. Add login route to `src/routes.tsx`
2. Create protected route component that checks auth session
3. Update `src/App.tsx` to wrap app with AuthProvider
4. Add auth gate - redirect unauthenticated users to login
5. Add logout button to header/sidebar

### Phase 3: Deploy to Vercel
1. Create Vercel project linked to GitHub repo
2. Configure environment variables (Supabase URL, anon key)
3. Add `vercel.json` config for SPA routing
4. Deploy and verify

## Notes
- Supabase free tier is sufficient for this use case
- Auth state persisted in localStorage via Supabase
- No changes to existing data layer needed (still uses JSON files)
