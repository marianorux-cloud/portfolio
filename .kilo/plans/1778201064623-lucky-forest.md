# Reservation Validation & Field Styling Review Plan

## Overview

This plan addresses validation issues and field styling inconsistencies in the add/edit reservation forms. The codebase has two reservation form implementations:
1. `ReservationForm.tsx` - Uses react-hook-form with zod schema
2. `reservations-edit-drawer.tsx` - Uses manual state management with custom validation

## Actual Issues Found (via code review)

### 1. Validation Not Using Schema (Critical)
**Location**: `src/features/reservations/reservations-edit-drawer.tsx`

**Problem**: The edit drawer uses manual validation with a simple `errors` state (`Record<string, boolean>`) instead of integrating with the existing `reservationSchema`. The schema exists at `src/lib/schemas/reservationSchema.ts` but is NOT used in the drawer.

**Current implementation**:
- Lines 99-131: Manual validation in `handleSubmit`
- Line 47: `const [errors, setErrors] = useState<Record<string, boolean>>({})`
- No field-level validation (real-time feedback)
- No integration with react-hook-form or zod
- "Other" location fields (`pickupLocationOther`, `dropoffLocationOther`) are NOT validated

### 2. Validation State Shape Issue
**Location**: `src/features/reservations/reservations-edit-drawer.tsx:47`

**Problem**: `errors` is `Record<string, boolean>` but error messages are hardcoded strings. The state only tracks if an error exists, not the message.

### 3. Input Error Border Color
**Location**: `src/components/ui/input.tsx:12`

**Problem**: Uses hardcoded `border-red-500` instead of theme token `border-destructive`.

**Current**:
```tsx
"aria-invalid:border-red-500 focus-visible:aria-invalid:ring-red-500"
```

**Should be**:
```tsx
"aria-invalid:border-destructive focus-visible:aria-invalid:ring-destructive"
```

### 4. Combobox Missing Error Support
**Location**: `src/components/ui/combobox.tsx`

**Problem**: Combobox has no `error` prop and no `aria-invalid` styling. The Select and DateRangePicker both have error state support, but Combobox does not.

**Current**:
- Line 66: No `aria-invalid` attribute
- Line 62-71: Button has no error styling

### 5. Combobox Placeholder Styling Inconsistency
**Location**: `src/components/ui/combobox.tsx:69`

**Problem**: Uses static `<span className="text-muted-foreground">` for placeholder, while Select uses `data-[placeholder]:text-muted-foreground` on SelectValue.

### 6. Combobox Padding Mismatch
**Location**: `src/components/ui/combobox.tsx:66`

**Problem**: Button has no explicit padding override, so it uses Button's default `px-4` instead of SelectTrigger's `px-3`.

### 7. TimePicker Already Integrated
**Note**: The edit drawer at lines 474 and 511 already uses `<TimePicker>`. The TimePicker component exists but lacks a clock icon.

## Detailed Fix Plan

### Phase 1: Input Error Styling (Quick Fix)
**File**: `src/components/ui/input.tsx:12`

Change:
```tsx
"aria-invalid:border-red-500 focus-visible:aria-invalid:ring-red-500"
```
To:
```tsx
"aria-invalid:border-destructive focus-visible:aria-invalid:ring-destructive"
```

### Phase 2: Combobox Component Improvements
**File**: `src/components/ui/combobox.tsx`

Add `error` prop and styling:
```tsx
interface ComboboxProps {
  // ... existing props
  error?: boolean;  // Add this
}

// In Button:
<Button
  variant="outline"
  role="combobox"
  aria-expanded={open}
  aria-invalid={error}
  className={cn(
    "w-full justify-between shadow-none bg-transparent px-3",
    error && "border-destructive focus-visible:ring-destructive",
    className
  )}
  disabled={disabled}
>
  {selectedOption ? selectedOption.label : (
    <span data-placeholder className="text-muted-foreground">{placeholder}</span>
  )}
  <ChevronDown className="ml-2 h-4 w-4 shrink-0 opacity-50" />
</Button>
```

### Phase 3: Validation Refactor - Option A Selected
**Chosen approach**: Integrate react-hook-form with zod (consistent with ReservationForm.tsx)

**Changes**:
1. Import `useForm` from react-hook-form
2. Import `zodResolver` from @hookform/resolvers/zod
3. Import `reservationSchema` from `@/lib/schemas/reservationSchema`
4. Replace manual state with `form` object
5. Use `FormField`, `FormItem`, `FormLabel`, `FormControl`, `FormMessage` components
6. Remove manual `errors` state

**Pros**:
- Consistent with `ReservationForm.tsx` implementation
- Real-time validation
- Schema-driven error messages
- Automatic error clearing

**Cons**:
- Larger refactor
- Need to handle Combobox integration with Controller

#### Option B: Improve Manual Validation (Rejected)
Keep manual state but improve the implementation:
1. Change `errors` type to `Record<string, string>` for messages
2. Add validation for "Other" location fields
3. Add `aria-invalid` to Combobox when error

### Phase 4: TimePicker Clock Icon
**File**: `src/components/ui/time-picker.tsx`

Add clock icon:
```tsx
import { Clock } from "lucide-react"

// Wrap Input:
<div className="relative">
  <Clock className="absolute left-3 top-1/2 -translate-y-1/2 h-4 w-4 text-muted-foreground pointer-events-none" />
  <Input
    type="time"
    value={value}
    onChange={(e) => onChange(e.target.value)}
    disabled={disabled}
    className={cn("pl-9", error && "border-destructive focus-visible:ring-destructive")}
  />
</div>
```
