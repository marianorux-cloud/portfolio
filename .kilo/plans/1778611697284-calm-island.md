# Card-Based Mobile Layout Plan

## Goal
Replace desktop table with card layout on mobile while preserving all column data and actions.

## Architecture

```
DataTable (existing)
├── Desktop: Table element (current)
└── Mobile: MobileCardView (new)
```

## Implementation Steps

### Phase 1: Create MobileCardView Component
**File**: `src/components/data-table/mobile-card-view.tsx`

- Accepts same `table` prop as DataTable
- Renders each row as a card with vertical layout
- Maps all visible columns to card fields
- Maintains row click behavior (opens drawer)
- Shows action buttons inline

### Phase 2: Define Column Rendering Strategy
**File**: `src/components/data-table/card-field-renderer.tsx`

- Generic field renderer that handles different column types
- Supports: string, number, date, badge, custom components
- Truncates long text with ellipsis

### Phase 3: Integrate with DataTable
**File**: `src/components/data-table/data-table.tsx`

- Import `useMobile` hook
- Conditionally render Table or MobileCardView
- Pass columns to MobileCardView for field mapping

### Phase 4: Mobile-Specific Styling
**File**: `src/components/data-table/mobile-card-view.tsx`

- Card: `bg-card border rounded-lg p-3 space-y-2`
- Field rows: `flex justify-between text-sm`
- Label: `text-muted-foreground`
- Value: `font-medium`
- Action buttons: horizontal flex, right-aligned

## Action Implementation

### Row Actions Component
**File**: `src/components/data-table/row-actions.tsx` (new)

- Accept `row`, `onView`, `onEdit`, `onDelete` props
- Render as horizontal button group on cards
- Hide on desktop (actions column handles this)

### Integration with Existing Actions
- Map `row.original` to existing action components
- Pass click handlers to drawer opener

## Responsive Breakpoints
- Mobile: `< 768px` (md breakpoint)
- Tablet: `768px - 1023px` (use table)
- Desktop: `>= 1024px` (use table)
